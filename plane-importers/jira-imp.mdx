## Overview

With the Jira importer, you can easily import issues, states, labels, priorities and user data from Jira to Plane. You can continue managing your existing projects and save time without having to re-enter data making your transition smooth and efficient.

## Import from Jira

To import Jira issues to a Plane project:

1. Go to the workspace option in Plane. Click **Workspace settings**.
2. Select **Imports** on the right pane. 
3. Click the **Import** button.
        <Frame>![Import from Jira](/images/importers/jira/import-from-jira.webp)</Frame>
4. Click the **Connect Jira** button on the **Jira to Plane Migration Assistant** page.
        <Frame>![Connect Jira](/images/importers/jira/connect-jira.webp)</Frame>
5. You will be prompted to sign in to your Atlassian account. Enter your credentials and sign in.
6. Click **Accept** to allow Plane access to your Atlassian account. If you have multiple workspaces with the same account, you will be asked to select the one from which you want to import the data to Plane in the **Use app on** dropdown list.
7. In the **Configure Plane** step, select the Plane project to which you want to import your Jira project issues. Click **Next Step**.
        <Frame>![Configure Plane](/images/importers/jira/configure-plane.webp)</Frame>
8. In the **Configure Jira** step:
    1. Select the workspace and the project from which you want to import in the **Select Jira Workspace** and **Select Jira Project** dropdown lists.
    2. For **How do you want to record issue types in Plane?**, select the option to define the way issue types in Jira should be displayed in the Plane project issues.
        - **Create as a label** - Issue types from Jira will be converted as Labels in the Plane project issues.
        - **Add [ issue_type ] in the title** - Issue types from Jira will appear in the title of the Plane project issues. 
    <Frame>![Configure Jira](/images/importers/jira/configure-jira.webp)</Frame>
10. In the **Import users** step, you have two options: 
    - **Upload CSV**: Click the **Upload CSV** button to import the users to your Plane project. Refer to [Export users from a site](https://support.atlassian.com/organization-administration/docs/export-users-from-a-site/) to download the CSV file from Jira. 
    - **Skip user import** - You can select the **Skip Importing User data** checkbox if you want to skip this step. You can manually add users after the import process. (not recommended)
        <Warning>If you skip user import, the issues and comments added by other users will by default show the name of the user who performed the migration. The Assignees field will be empty.</Warning>
    <Frame>![Import users](/images/importers/jira/import-users.webp)</Frame>
10. In the **Map states** step:
    - Map the **Jira status** to their equivalent **Plane states**.
    - Select the **Auto create and map the remaining Jira states** checkbox to automatically create and map the missing states from Jira in Plane.
    <Frame>![Map states](/images/importers/jira/map-states.webp)</Frame>
11. In the **Map priorities** step, map the **Jira priorities** to their equivalent **Plane priorities**. If a match doesn't exist, select **None** in the Plane priority list.
    <Frame>![Map priorities](/images/importers/jira/map-priorities.webp)</Frame>
12. In the **Summary** step, review the mappings and make any changes, if needed. Click **Confirm** to start migration.
    <Frame>![Review mappings](/images/importers/jira/review-mappings.webp)</Frame>
13. The data migration begins and takes a few minutes to complete depending on the number of issues in your Jira workspace. 
    <Frame>![Migration complete](/images/importers/jira/import-complete.webp)</Frame>
14. Navigate to **Issues** in your project on the sidebar to verify that the data import is successful. 

## Sync Jira to Plane

After import, if a new issue is created or an existing issue is updated in Jira, you can easily migrate these changes to your Plane project. Follow the steps below:

1. Go to the workspace option in Plane. Click **Workspace settings**.
2. Select **Imports** on the right pane. 
3. Click the **Re run** button
